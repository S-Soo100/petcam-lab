# 라벨링 웹 사람 판정 문구·촬영 환경 UX 개선 설계

> 상태: **구현 완료 · 검증 완료** (2026-07-14)
> 대상: 라벨링 웹 `1단계 · 사람 판정`의 production·tutorial 공유 폼
> 구현 담당: Claude
> 핵심 제약: 다른 세션이 수정 중인 실험 문서와 `specs/next-session.md`를 건드리지 않는다.
> 검증: web Vitest 263개, Python pytest 334개, TypeScript, diff check, Next.js 14.2.35 production build 통과

## 1. 배경

실제 라벨러 사용에서 다음 문제가 확인됐다.

1. `촬영 환경`이 필수 확인 항목인지 화면에 표시되지 않는다. 영상에 따라 고를 환경이 하나도 없어서, 빈 값이 "확인 후 해당 없음"인지 "선택 누락"인지 구분되지 않는다.
2. 대표 행동 대상의 `사물`과 `도구`가 단어만으로 노출돼 자주 혼동된다. 여기서 `도구`는 주로 사람이 직접 먹일 때 사용하는 급여 도구를 뜻한다.
3. `휴식/바스킹`은 실제로 게코가 가만히 있고 그림자나 조명만 변하는 영상에서 선택된다. 이를 `일광욕`으로 좁히면 실제 라벨링 기준과 어긋난다.
4. `페이스트 먹기`, `먹이 사냥/섭취`는 실제 운영 용어와 맞지 않는다.

이번 작업은 저장 enum을 바꾸는 라벨 온톨로지 마이그레이션이 아니다. 기존 데이터 계약을 유지하면서 라벨러가 화면에서 같은 기준으로 판단하도록 문구와 선택 상태를 명확히 하는 UX 개선이다.

## 2. 목표

- 라벨러가 `촬영 환경`을 반드시 확인하고, 해당 항목이 없으면 `해당 없음`을 직접 선택한다.
- `사물`과 `도구`를 사용 목적과 예시로 구분한다.
- 게코가 실제로 움직인 장면과 그림자·조명만 변한 휴식 장면을 구분한다.
- production과 tutorial의 공유 사람 판정 폼에서 같은 문구와 동작을 사용한다.
- 기존 GT, API, DB, tutorial seed와 호환성을 유지한다.

## 3. 제외 범위

- `PrimaryAction`, `Target`, `ContextTag` enum 추가·삭제·이름 변경
- DB 마이그레이션 또는 Supabase 데이터 수정
- API payload·응답 형식 변경
- 기존 tutorial lesson seed·활성 버전·진행 이력 변경
- VLM 프롬프트, classifier class 정의, Flutter 문구 변경
- 기존 GT의 일괄 데이터 변환
- `specs/next-session.md`, `experiments/`, 현재 다른 세션의 handoff 문서 수정

## 4. 채택한 접근

### 4.1 촬영 환경의 `해당 없음`은 UI 전용 상태로 둔다

`ContextTag`에 `none`을 추가하지 않는다. 화면에서 `해당 없음`을 직접 선택하면 기존 저장값인 `context_tags: []`를 사용한다.

이 방식은 다음 두 상태를 화면에서 구분한다.

- 아직 확인하지 않음: `explicitlySelected`에 `context_tags`가 없고, 어떤 버튼도 활성화되지 않음
- 확인했지만 해당 항목 없음: `explicitlySelected`에 `context_tags`가 있고, `context_tags`는 빈 배열이며 `해당 없음`이 활성화됨

서버는 기존처럼 빈 배열을 허용한다. 직접 선택 필수 규칙은 기존 visibility·primary action·highlight와 같은 client UX 계약으로만 강제한다.

### 4.2 화면 문구만 바꾸고 저장 enum은 유지한다

| 저장 enum | 기존 화면 문구 | 새 화면 문구 |
|---|---|---|
| `basking` | 휴식/바스킹 | **휴식** |
| `eating_paste` | 페이스트 먹기 | **슈퍼푸드 자율급여** |
| `eating_prey` | 먹이 사냥/섭취 | **곤충 사냥** |
| `tool` | 도구 | **급여 도구 (숟가락·주사기·핀셋 등)** |
| `object` | 사물 | **일반 사물 (장식물·은신처·나뭇가지 등)** |

`basking`은 호환성을 위한 저장 키로 남긴다. 라벨링 웹의 사람 판정 화면에서는 열원 아래 일광욕으로 좁히지 않고, **게코가 실질적으로 움직이지 않고 쉬는 상태**를 뜻한다. classifier나 다른 문서의 `basking` 의미까지 이번 작업에서 암묵적으로 바꾸지 않는다.

## 5. 사용자 체험 설계

### 5.1 촬영 환경을 확인하는 흐름

`[화면]` 라벨러는 `촬영 환경 (필수)` 제목과 "영상에 해당하는 환경을 모두 골라줘. 해당하는 항목이 없으면 ‘해당 없음’을 선택해." 안내를 본다.

`[조작]` 라벨러가 하나 이상의 환경 태그를 누르거나 `해당 없음`을 누른다.

`[반응]` 환경 태그와 `해당 없음`은 상호 배타적으로 표시된다. 선택하지 않고 저장하면 촬영 환경으로 스크롤되고 오류가 보인다.

`[감정]` 라벨러는 항목을 빠뜨린 것이 아니라 영상을 확인하고 환경 이상이 없다고 명시했다는 확신을 갖는다.

### 5.2 그림자·조명 변화만 있는 휴식 영상

`[화면]` 영상에는 그림자나 조명 변화가 있지만 게코의 위치와 자세에는 실질적인 움직임이 없다.

`[조작]` 라벨러가 대표 행동 `휴식`을 누른다.

`[반응]` 도움말이 "그림자나 조명만 변하고 게코가 움직이지 않았다면 휴식"이라고 판정 기준을 확인시킨다.

`[감정]` 라벨러는 `일반 이동`을 억지로 선택하지 않고 실제 개체 행동과 영상 노이즈를 구분한다.

### 5.3 급여 도구와 일반 사물 구분

`[화면]` 대표 행동 대상 선택 영역에서 사용 목적과 예시가 포함된 두 선택지를 본다.

`[조작]` 사람이 먹이를 전달할 때 쓴 숟가락·주사기·핀셋이면 `급여 도구`, 사육장 안 구조물이나 장식이면 `일반 사물`을 고른다.

`[반응]` 선택 영역의 안내문이 두 범주의 경계를 다시 보여준다.

`[감정]` 라벨러는 "둘 다 물건인데 무엇을 골라야 하나"를 추측하지 않는다.

## 6. 정확한 화면 문구

### 6.1 촬영 환경

제목:

> 촬영 환경 (필수)

안내문:

> 영상에 해당하는 환경을 모두 골라줘. 해당하는 항목이 없으면 ‘해당 없음’을 선택해.

신규 UI 전용 선택지:

> 해당 없음

미선택 저장 오류:

> 촬영 환경을 골라줘. 해당하는 항목이 없으면 ‘해당 없음’을 선택해.

### 6.2 대표 행동과 도움말

`휴식` 도움말:

> 게코가 위치를 옮기거나 뚜렷한 행동을 하지 않고 가만히 쉬는 장면. 그림자나 조명만 변하고 게코가 움직이지 않았다면 휴식이야.

`일반 이동` 도움말:

> 게코가 실제로 위치를 옮기거나 등반하거나 자세를 바꾸는 장면. 그림자나 조명만 변하고 게코가 가만히 있으면 휴식이야.

`슈퍼푸드 자율급여` 도움말:

> 그릇에 제공된 슈퍼푸드를 게코가 스스로 핥아 먹는 장면.

`곤충 사냥` 도움말:

> 게코가 곤충을 추적·포획하거나 삼키는 장면.

### 6.3 사물과 도구

선택지:

- `급여 도구 (숟가락·주사기·핀셋 등)`
- `일반 사물 (장식물·은신처·나뭇가지 등)`

대표 행동 대상 영역의 보조 안내문:

> 먹이를 직접 전달하는 데 사용한 물건은 ‘급여 도구’, 사육장 안의 구조물이나 장식은 ‘일반 사물’을 골라줘.

기존 `사람이 직접 먹임`의 동적 질문은 유지하되 설명을 다음처럼 맞춘다.

> 손을 사용해 직접 먹였는지, 급여 도구를 사용해 먹였는지 골라.

## 7. 상태와 상호작용 계약

### 7.1 새 GT

1. 최초 진입 시 기존 환경 태그와 `해당 없음` 모두 비활성이다.
2. 환경 태그를 하나 누르면 해당 태그를 토글하고 `context_tags`를 직접 선택한 필드로 기록한다.
3. 마지막 환경 태그를 해제해 빈 배열이 되면 `해당 없음`이 활성화된다. 라벨러가 직접 환경 영역을 조작했으므로 확인 완료 상태다.
4. `해당 없음`을 누르면 모든 환경 태그를 지우고 빈 배열로 만든다.
5. `해당 없음` 활성 상태에서 환경 태그를 누르면 해당 없음 상태가 풀리고 그 태그가 활성화된다.
6. 촬영 환경을 한 번도 조작하지 않고 저장하면 client validation이 저장을 막는다.

### 7.2 기존 GT와 보정 화면

- 저장된 GT를 다시 열 때는 기존 `allSelectedFields()` 계약을 유지한다.
- 저장된 `context_tags: []`는 `해당 없음` 활성 상태로 표시한다.
- 저장된 환경 태그가 있으면 해당 태그만 활성화하고 `해당 없음`은 비활성이다.
- 기존 GT를 읽기 위해 데이터 변환이나 마이그레이션을 하지 않는다.

### 7.3 사람이 직접 먹임과의 관계

- 대표 행동이 `hand_feeding`이면 기존 규칙대로 `context_tags`에 `human`이 필요하다.
- 이 상태에서 `해당 없음`을 고르면 촬영 환경 직접 선택 조건은 만족하지만, hand-feeding 의미 검증은 실패해야 한다.
- 기존 오류 위치와 `사람 등장` 안내를 유지한다. `해당 없음` 추가로 hand-feeding 검증을 약화하지 않는다.

## 8. 표시·데이터 경계

- 한국어 화면 문구는 기존 공통 표시 계층을 단일 진실 공급원으로 사용한다.
- API payload에는 기존 enum인 `basking`, `eating_paste`, `eating_prey`, `tool`, `object`를 보낸다.
- `해당 없음`은 `ContextTag`가 아니며 API payload에 문자열로 보내지 않는다.
- `context_tags: []`의 서버 유효성 계약은 유지한다.
- 라벨 요약, 튜토리얼, 보정 화면처럼 공통 formatter를 쓰는 화면에도 새 한국어 명칭이 일관되게 보여야 한다.
- raw enum을 사람이 보는 새 문구로 치환하는 과정에서 enum 자체를 rename하지 않는다.

## 11. 검증 기준

### 11.1 단위 테스트

- `ACTION_LABELS.basking === '휴식'`
- `ACTION_LABELS.eating_paste === '슈퍼푸드 자율급여'`
- `ACTION_LABELS.eating_prey === '곤충 사냥'`
- `TARGET_LABELS.tool`에 `급여 도구`, `숟가락`, `주사기`, `핀셋`이 포함된다.
- `TARGET_LABELS.object`에 `일반 사물`과 사육장 사물 예시가 포함된다.
- 직접 선택 집합에 `context_tags`가 없으면 `context_tags_not_selected` 오류가 발생한다.
- 직접 선택 집합에 `context_tags`가 있고 값이 `[]`이면 ordinary GT가 통과한다.
- `hand_feeding`에서 값이 `[]`이면 기존 `hand_feeding_context` 오류가 계속 발생한다.
- server validation은 `context_tags: []`를 계속 허용한다.

### 11.2 화면 확인

- 새 GT 최초 화면에서 `해당 없음`이 미리 선택돼 있지 않다.
- 촬영 환경을 건너뛰고 저장하면 해당 영역으로 이동하고 정확한 오류를 보여준다.
- `해당 없음`과 개별 태그가 동시에 활성화되지 않는다.
- 기존 빈 배열 GT를 열면 `해당 없음`이 활성화된다.
- production과 tutorial 사람 판정 폼에서 같은 문구와 동작을 보인다.
- 모바일 폭에서 긴 `급여 도구`·`일반 사물` 옵션이 잘리거나 선택 불가능해지지 않는다.

### 11.3 회귀 검증

Claude는 레포의 `uv`·`web` 실행 규칙과 현재 package script를 실제로 확인한 뒤 관련 Vitest, TypeScript 검사, production build를 실행한다. 기억으로 명령을 만들지 않는다. 실패하면 원인을 진단하고 최대 3회 수정 루프 안에서 해결한다.

## 12. 완료 조건

- 촬영 환경이 필수 확인 항목임이 제목과 안내문에 드러난다.
- `해당 없음`을 직접 선택할 수 있고 기존 빈 배열로 저장된다.
- 미확인과 확인 후 해당 없음이 화면 상태에서 구분된다.
- 사물과 급여 도구가 명칭·예시·안내문으로 구분된다.
- `휴식`, `슈퍼푸드 자율급여`, `곤충 사냥`이 사람에게 보이는 공통 명칭이 된다.
- 휴식과 일반 이동 도움말이 그림자·조명 변화와 실제 게코 이동을 구분한다.
- DB·API·enum·tutorial seed·기존 GT 데이터는 변경되지 않는다.
- 관련 테스트와 build가 통과한다.
- 다른 세션의 미커밋 파일을 수정하거나 stage하지 않는다.
