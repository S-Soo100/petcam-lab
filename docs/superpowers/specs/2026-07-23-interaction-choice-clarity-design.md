# 놀이 상호작용 선택지 직관화 설계

**상태:** 사용자 방향 승인 · 구현 전 설계 고정
**작성:** 2026-07-23
**대상:** `GroundTruthForm`의 `enrichment_object` / `interaction_types` 입력 UI

## 1. 문제

현재 `올라타기 / 밀기 / 회전시키기 / 쫓기 / 반복해서 돌아오기 / 기타`가 설명 없는 작은 칩으로 나열된다. 라벨러는 서로 배타적인 선택인지, 쳇바퀴에서 무엇을 골라야 하는지, `밀기`와 `회전시키기`를 동시에 골라도 되는지 알기 어렵다. 위 안내문에 단어를 반복하는 것만으로는 판단 기준이 생기지 않는다.

## 2. 목표

- 선택지를 읽는 순간 **관찰 기준**이 보이게 한다.
- 쳇바퀴에서는 자주 쓰는 `ride / push / rotate`를 먼저 보여준다.
- 복수 선택임을 명시하고 선택 결과를 자연어로 다시 확인시킨다.
- 저장 enum·API·DB·기존 GT는 변경하지 않는다.

## 3. 대안 비교

1. **설명 툴팁:** 화면은 작지만 라벨러가 항목마다 다시 눌러야 해 실제 작업 중 읽히지 않는다.
2. **긴 공통 안내문:** 구현은 작지만 여섯 항목의 경계가 여전히 선택지와 떨어져 있다.
3. **설명 내장 선택 카드 + 상황별 그룹 + 선택 요약 — 채택:** 한 번에 판단 기준이 보이고 모바일에서도 항목별 의미가 유지된다.

## 4. 사용자 체험

1. **[화면]** 라벨러가 `쳇바퀴`를 선택한다.
   **[반응]** 질문이 `쳇바퀴에서 실제로 본 행동은?`으로 바뀌고 `여러 개 선택할 수 있어`가 바로 보인다.
   **[감정]** 무엇을 답해야 하는지 즉시 안다.

2. **[화면]** 먼저 세 개의 큰 선택 카드가 보인다.
   - `위·안에 올라감` — 몸이나 발을 쳇바퀴 위 또는 안에 올렸어요.
   - `밖에서 밀거나 건드림` — 올라타지 않고 발·머리·몸으로 쳇바퀴를 밀었어요.
   - `쳇바퀴를 실제로 돌림` — 게코의 움직임 때문에 쳇바퀴가 회전했어요.
   **[조작]** 실제로 본 항목을 모두 선택한다.
   **[반응]** 선택 카드는 보라색 테두리·배경과 체크 표시로 명확히 활성화된다.

3. **[화면]** 아래 `그 밖에 함께 본 행동`에 드문 선택지가 분리된다.
   - `움직이는 물체를 따라감` — 돌아가거나 움직이는 물체를 쫓아갔어요.
   - `떠났다가 다시 찾아옴` — 다른 곳에 갔다가 같은 물체로 여러 번 돌아왔어요.
   - `다른 방식으로 상호작용함` — 위 설명에는 없지만 물체를 분명히 사용했어요.

4. **[반응]** 선택 직후 `선택한 내용: 쳇바퀴 위·안에 올라감 · 쳇바퀴를 실제로 돌림`처럼 자연어 요약이 보인다.
   **[감정]** 저장 전에 자신의 판단을 다시 확인할 수 있다.

## 5. UI 계약

### 5.1 제목과 그룹

- `wheel` 선택 시 제목: **쳇바퀴에서 실제로 본 행동은?**
- 그 외 사물 선택 시 제목: **이 물체를 어떻게 사용했나?**
- 공통 보조문: **여러 개 선택할 수 있어. 실제로 확인한 행동을 모두 골라줘.**
- `wheel`에서 `ride / push / rotate`는 `자주 확인하는 행동`, 나머지는 `그 밖에 함께 본 행동`으로 분리한다.
- 다른 사물에서도 같은 카드와 설명을 사용하되 하나의 목록으로 보여준다. 저장값 의미는 사물 종류와 무관하게 유지한다.

### 5.2 표시 문구와 저장값

| 저장값 | 선택 카드 제목 | 설명 |
|---|---|---|
| `ride` | 위·안에 올라감 | 몸이나 발을 물체 위 또는 안에 올렸어요. |
| `push` | 밖에서 밀거나 건드림 | 올라타지 않고 발·머리·몸으로 물체를 밀었어요. |
| `rotate` | 물체를 실제로 돌림 | 게코의 움직임 때문에 물체가 회전했어요. |
| `chase` | 움직이는 물체를 따라감 | 돌아가거나 움직이는 물체를 쫓아갔어요. |
| `repeated_return` | 떠났다가 다시 찾아옴 | 다른 곳에 갔다가 같은 물체로 여러 번 돌아왔어요. |
| `other` | 다른 방식으로 상호작용함 | 위 설명에는 없지만 물체를 분명히 사용했어요. |

`wheel`에서는 제목·설명의 `물체`를 `쳇바퀴`로 바꿔 보여준다. 저장 payload는 기존 enum 배열 그대로다.

### 5.3 레이아웃·접근성

- 모바일 1열, `sm` 이상 2열 카드.
- 카드 전체가 `<button type="button">`이며 선택 여부를 `aria-pressed`로 제공한다.
- 제목은 본문보다 한 단계 진하고, 설명은 작은 회색 텍스트로 둔다.
- 아이콘만으로 의미를 전달하지 않는다. 체크 표시는 선택 상태 보조일 뿐이다.
- 오류 문구는 기존 `FieldError(interaction_types)`를 그대로 사용한다.

## 6. 구현 경계

### 포함

- `labelingDisplay.ts`에 화면용 제목·설명·그룹·선택 요약 순수 계약 추가.
- `_labeling-forms.tsx`의 interaction chip을 설명 카드로 교체.
- 공유 `GroundTruthForm`을 쓰는 legacy v2·motion v3·튜토리얼·보정 화면에 일관되게 적용.
- 표시 계약과 그룹·요약에 대한 단위 테스트.

### 제외

- `InteractionType` enum, API, RPC, migration, DB 변경.
- 이미 저장된 GT 변환.
- 영상 예시·썸네일·애니메이션 추가.
- `playing` 자동 판정 또는 selector/VLM/Python Evidence 변경.

## 7. 테스트·완료 조건

1. RED: 여섯 저장값에 제목·설명이 없으면 실패한다.
2. RED: wheel 그룹이 `ride/push/rotate` 우선 순서를 지키지 않으면 실패한다.
3. RED: 선택 요약이 wheel 문맥과 복수 선택을 자연어로 만들지 못하면 실패한다.
4. GREEN: 기존 클릭 시 배열 toggle 의미와 payload가 byte-equivalent다.
5. web 전체 테스트·TypeScript·Next build 통과.
6. preview에서 wheel 선택 후 카드 6개·복수 선택·선택 요약·모바일 1열을 확인한다.

## 8. 성공 기준

라벨러가 별도 설명을 묻지 않고 다음 세 가지를 구분할 수 있어야 한다.

- 올라갔지만 안 돌았음 → `ride`
- 밖에서 건드려 돌림 → `push + rotate`
- 올라가 걸으며 돌림 → `ride + rotate`
