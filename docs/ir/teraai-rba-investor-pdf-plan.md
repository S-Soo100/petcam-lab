# TeraAI RBA 투자자 PDF용 압축본 기획서

## 1. 문서 목적

투자자가 PDF만 읽어도 TeraAI의 RBA가 단순 펫캠 기능이나 고정 알고리즘이 아니라, 파충류 행동 데이터를 해석하고 축적하는 AI 기술 자산이라는 점을 이해하게 만든다.

## 2. 커뮤니케이션 목표

**By the end, 투자자는 TeraAI가 밤사이 파충류 펫캠 영상을 행동 타임라인과 케어 신호로 바꾸는 AI 분석 엔진을 보유하고 있다고 믿어야 한다. 왜냐하면 RBA는 영상 이해 AI, 사육 도메인 규칙, 사람 검수 데이터 루프를 결합해 시간이 지날수록 더 강해지는 구조이기 때문이다.**

## 3. 핵심 메시지

TeraAI의 RBA는 단순 움직임 감지나 고정 알고리즘이 아니라, 밤사이 영상을 행동 타임라인과 케어 신호로 바꾸는 파충류 특화 AI 분석 엔진이다.

## 4. 투자자가 가져가야 할 믿음

1. 이것은 단순 카메라 기능이 아니라 AI 기술 자산이다.
2. Track A / Track B 구조는 비용과 정확도 문제를 동시에 다루는 현실적인 AI 운영 구조다.
3. 데이터가 쌓일수록 도마뱀 특화 모델과 사육 규칙이 강해진다.
4. 장기적으로 영상 데이터, 행동 라벨, 사육 맥락이 결합되어 복제 난이도가 생긴다.

## 5. 전체 구성 원칙

- 본문은 7장으로 압축한다.
- 상세 기술 구조는 부록 3장으로 분리한다.
- 투자자용 PDF는 발표자가 없어도 읽혀야 하므로, 각 장표 제목을 주제명이 아니라 주장형 문장으로 쓴다.
- 기술명은 첫 등장 때 쉬운 말을 먼저 쓰고, 괄호 안에 기술명을 붙인다.
- 한 장에는 개념 하나와 예시 하나만 둔다.
- "AI다"라는 주장은 기능명이 아니라 판단 구조와 데이터 루프로 증명한다.

## 6. 권장 용어 변환

| 기존 용어 | 투자자용 표현 | 설명 방향 |
|---|---|---|
| RBA | 파충류 행동 분석 AI | 밤사이 영상을 행동 일지로 바꾸는 엔진 |
| Track A | 빠른 1차 판독 AI | 모든 움직임 클립을 저비용으로 먼저 보는 단계 |
| Track B | 정밀 판독 AI | 애매하거나 중요한 장면을 증거와 함께 다시 보는 단계 |
| confidence | AI 자기확신 점수 | 정답 확률이 아니라, 정밀 분석으로 보낼지 판단하는 신호 |
| ROI | 관심 구역 | 먹이그릇, 물그릇, 은신처처럼 판단에 중요한 위치 |
| HITL | 사람 검수 루프 | 보호자/운영자가 고친 답이 다시 데이터가 되는 구조 |
| structured context packet | 판단 재료 묶음 | 시간, 위치, 움직임, 종 정보, 전후 맥락을 묶어 AI에게 주는 방식 |
| VLM | 영상을 이해하는 AI | 이미지를 보고 의미를 해석하는 범용 시각 AI |

## 7. 본문 7장 구성

### Slide 1. 우리는 밤사이 영상을 행동 일지로 바꾼다

**목적:** RBA가 무엇을 하는지 한 문장으로 이해시킨다.

**핵심 문장:**  
TeraAI는 밤사이 펫캠 영상을 AI가 읽을 수 있는 행동 타임라인과 케어 신호로 바꾼다.

**넣을 요소:**
- 카메라 / 움직임 클립 / AI 분석 / 행동 일지 / 케어 알림 흐름
- 기존 1번 전체 구조도와 2번 수집 파이프라인을 압축
- 세부 기술명보다 "결과물"을 먼저 보여준다.

**줄일 요소:**
- Supabase, R2 Storage 같은 내부 구현명은 본문에서 삭제하거나 아주 작게 처리
- Track A / Track B 상세 흐름은 뒤 장표로 이동

### Slide 2. 실제 장면을 보면 기술이 바로 이해된다

**목적:** 기술 설명 전에 실제 데이터가 어떤 모습인지 보여준다.

**핵심 문장:**  
RBA는 도마뱀의 실제 행동 장면을 보고, 행동 라벨과 판단 근거를 함께 남긴다.

**넣을 요소:**
- eating_paste, eating_prey, hand_feeding 실제 이미지 3개
- 각 이미지 아래 한 줄 판정 기준
- 현재 7번 슬라이드를 앞으로 이동

**권장 카피:**
- 사료 섭취: 그릇이 보이고 혀가 먹이에 반복 접촉
- 먹이 사냥: 먹이가 보이고 몸 방향과 시선이 고정
- 사람 급여: 손이나 도구가 먹이를 직접 전달

### Slide 3. Track A는 모든 움직임을 빠르게 1차 판독한다

**목적:** Track A의 역할과 confidence를 쉽게 설명한다.

**핵심 문장:**  
Track A는 모든 움직임 클립을 빠르게 훑고, 행동 라벨과 AI 자기확신 점수를 남긴다.

**넣을 요소:**
- 행동 라벨 7종
- 출력 형식: `action + confidence + reasoning`
- confidence 단계 3개

**주의 문장:**  
confidence는 정답 확률이 아니라 AI가 스스로 판단한 확실성 신호다. 이 점수는 Track B나 사람 검수로 보낼지 판단하는 데 쓴다.

### Slide 4. 프롬프트는 AI에게 주는 행동 판독 체크리스트다

**목적:** "프롬프트"가 단순 문장이 아니라 판독 기준이라는 점을 설명한다.

**핵심 문장:**  
RBA의 프롬프트는 AI가 행동을 과잉 해석하지 않도록 증거 기준을 정해주는 판독 매뉴얼이다.

**넣을 요소:**
- 입력: 60초 움직임 클립
- 판독 기준: 행동 라벨 7종
- 결정 규칙 예시
- 출력: JSON 결과

**예시 문장:**
- 그릇 근처에 있다고 먹었다고 보지 않는다.
- paste 섭취는 혀와 먹이의 반복 접촉이 필요하다.
- 음수는 몸을 고정하고 한 지점을 반복해서 핥는 자세를 본다.
- 모호하면 specialized 행동보다 moving을 우선한다.

### Slide 5. Track B는 애매한 장면을 증거와 함께 다시 본다

**목적:** Track B가 왜 필요한지 실제 사례로 납득시킨다.

**핵심 문장:**  
Track B는 애매한 장면을 시간, 위치, 움직임 같은 판단 재료와 함께 다시 분석한다.

**넣을 요소:**
- Track B 개념: 짧은 이벤트 조각, ROI, motion metadata
- drinking 회복 사례
- "Track A 오판 후보 -> Track B 정밀 분석 -> 최종 drinking" 흐름

**권장 구성:**
- 왼쪽: drinking key frame
- 오른쪽: 3단계 흐름
  1. Track A: eating_paste 후보, confidence 낮음
  2. Track B: 물그릇 ROI, 머리 위치, 혀 동작 재확인
  3. 최종: drinking

### Slide 6. RBA는 데이터가 쌓일수록 더 좋아지는 구조다

**목적:** 기술이 일회성 PoC가 아니라 누적 자산이라는 점을 보여준다.

**핵심 문장:**  
RBA는 행동 라벨과 사람 검수 데이터가 쌓일수록 도마뱀 특화 AI로 진화한다.

**넣을 요소:**
- RBA 1.0: Track A + Track B 베타
- RBA 1.5: 모호/중요 행동 선별적 정밀 분석
- RBA 2.0: 자체 서버 + 특화 모델 + 증거 기반 앙상블
- 다종 확장: 크레스티드 게코에서 다른 파충류/양서류로 확대

**표현 방향:**
- "로드맵"보다 "AI 자산 축적 단계"로 보여준다.

### Slide 7. 우리의 해자는 영상 AI, 특화 모델, 사육 규칙의 결합이다

**목적:** 왜 경쟁자가 쉽게 따라오기 어려운지 정리한다.

**핵심 문장:**  
외부 AI 하나를 쓰는 것이 아니라, 영상 AI와 우리 데이터, 사육 규칙을 결합하기 때문에 시간이 갈수록 복제 난이도가 올라간다.

**넣을 요소:**
- VLM: 영상을 이해하는 범용 시각 AI
- 특화 모델: 라벨 데이터로 학습하는 도마뱀 행동 AI
- 사육 규칙: 도메인 기준으로 AI 오판 보정
- 판단 재료 묶음: 시간, ROI, 움직임, 전후 맥락

**권장 마무리 문장:**  
RBA의 핵심 자산은 모델 하나가 아니라, 행동 데이터와 사육 맥락이 함께 쌓이는 판단 구조다.

## 9. 삭제하거나 축소할 내용

- 내부 인프라 이름: Supabase, R2 Storage 등은 투자자 본문에서 중요도가 낮다.
- `structured context packet`이라는 원어 표현은 본문에서 1차 노출하지 않는다.
- confidence 숫자 구간은 유지하되, 너무 기술적으로 보이면 본문에서는 3단계 카드로만 표현한다.
- 전체 관계도는 첫 장에서 너무 복잡하므로 본문에서는 단순 흐름으로 바꾸고 상세도는 부록으로 보낸다.
- "모든 영상을 AI에게 전달"은 별도 장표로 유지하지 않고 1번 슬라이드 안으로 흡수한다.

## 10. 최종 PDF 톤

- 설명서는 아니고 투자 판단을 돕는 기술 소개서여야 한다.
- "우리가 이런 기술을 연구했다"보다 "이 기술이 왜 사업 자산이 되는가"를 앞세운다.
- 중학생도 이해할 수 있는 쉬운 표현을 쓰되, 투자자가 볼 때 기술 깊이가 없어 보이면 안 된다.
- 쉬운 말과 기술명을 병기한다.
  - 예: 빠른 1차 판독 AI(Track A)
  - 예: 판단 재료 묶음(structured context packet)
