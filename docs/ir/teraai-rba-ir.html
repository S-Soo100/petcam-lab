RBA Track A의 행동 분석 프롬프트 (v4.0)란?
자체 개발·검증한 AI 행동 분석 엔진의 안정적인 1차 판독 기준선
  • 카메라가 밤새 촬영한 영상 중 움직임이 있는 60초 클립을 최신 멀티모달 VLM이 분석하고, 자체 프롬프트(v4.0)가 "어떤 행동으로 판단할지"를 통제
  • 결과는 먹이 반응·음수·탈피·이동·보이지 않음 같은 행동 라벨과 판단 신뢰도로 저장
  • 핵심은 단순히 "맞혀봐"가 아니라, 행동을 과잉 해석하지 않도록 증거 기준을 강하게 정의한다는 것
  • 예: 도마뱀이 그릇 근처에 있다고 바로 먹었다고 판단하지 않고, 실제 혀 접촉이나 반복적 섭취 행동이 보여야만 feeding으로 분류
  • 현재 프롬프트(v4.0)는 자체 평가셋 기준 약 89% 정확도를 달성한 1차 판독 기준선으로 고정 — 모델을 교체해도 동일 수준 정확도 유지 (특정 벤더에 종속되지 않음)

structured context packet 설계 ★ 승부처

✕ 나쁜 입력
"이 영상에서 도마뱀이 뭐 하고 있어?"
✓ 좋은 입력
"이 클립은 03:27에 발생했고, 물그릇 ROI 근처에서 motion burst가 있었고, 이 개체는 크레스티드 게코이며, 최근 2일간 음수 이벤트가 없었고, 전후 10초 프레임에서 머리가 물그릇 방향으로 이동했다. 이 증거를 기준으로 drinking / moving / unknown 중 판단해."
정리 지난 연구에서 이 가설을 직접 검증했다. 프롬프트만 정교하게 다듬어서는 넘지 못하는 한계가 분명히 존재했고, 모델에게 '볼 수 있는 증거'를 늘려줬을 때 비로소 정확도가 올라갔다. 즉 RBA의 다음 도약은 더 좋은 프롬프트가 아니라, 영상 주변의 증거를 구조화해 결합하는 데 있다 — 이것이 범용 AI를 그대로 가져다 쓰는 경쟁자가 따라오기 어려운 지점이다.
