# TeraAI RBA AI 쉬운 용어 정리집

기준 자료: `teraai-rba-ir.pdf` / 2026-06-22 / 9 slides

목적: 투자자가 AI 기술 용어를 몰라도 RBA가 무엇을 하는지 이해하게 만드는 설명서.

읽는 법:
- 어려운 영어 용어는 외우는 단어가 아니라, 투자자가 질문할 때 쉽게 풀어 말하기 위한 단어다.
- 핵심은 "밤새 찍힌 펫캠 영상을 AI가 행동 일지와 케어 신호로 바꾼다"는 것이다.
- 너무 기술적으로 설명하기보다 "사람이 영상을 돌려보며 판단하는 일을 AI가 구조적으로 대신한다"고 말하면 쉽다.

---

## 한 문장 요약

RBA는 밤새 찍힌 도마뱀 펫캠 영상에서 움직임이 있는 장면을 골라내고, AI가 행동을 분류하고, 애매한 장면은 더 자세히 분석해서 보호자에게 "어젯밤에 무엇을 했는지" 알려주는 행동 분석 엔진이다.

| Capture Worker | 실시간 영상을 짧은 영상 파일로 잘라 저장하는 작업자 프로그램 | AI가 볼 수 있는 분석 단위로 영상을 정리함 | "카메라 영상을 사람이 보기 좋은 짧은 클립으로 자동 분리하는 역할입니다." |
| 1분 mp4 | 1분 단위로 잘라 저장한 영상 파일 | 너무 긴 영상을 다루지 않고 관리하기 쉬워짐 | "밤새 긴 영상을 1분짜리 조각으로 나눠 분석 비용과 저장을 관리합니다." |
| Motion Detection | 움직임 감지 | 움직임 없는 장면은 버리고 중요한 장면만 남김 | "AI가 모든 정적 영상을 다 보는 게 아니라, 움직임이 있는 장면만 먼저 골라냅니다." |
| clip | 분석하기 좋게 잘라낸 짧은 영상 | RBA가 실제로 분석하는 기본 단위 | "긴 녹화본 전체가 아니라 짧은 클립 단위로 AI가 판단합니다." |
| clip metadata | 클립에 붙은 설명 정보 | 촬영 시간, 저장 위치, 분석 결과 등을 관리함 | "영상 조각마다 시간과 분석 결과 같은 꼬리표를 붙여 관리합니다." |
| Supabase DB | 클립 정보와 분석 결과를 저장하는 데이터베이스 | 앱에서 결과를 빠르게 불러오기 위한 기반 | "영상 자체가 아니라 영상의 정보와 AI 결과를 저장하는 장부입니다." |
| R2 Storage | 영상 파일과 썸네일을 보관하는 저장소 | 큰 영상 파일을 따로 안정적으로 보관함 | "영상 파일은 전용 창고에 보관하고, 앱은 필요한 장면만 꺼내 씁니다." |
| 썸네일 | 영상을 대표하는 작은 미리보기 이미지 | 사용자가 하이라이트를 빠르게 확인할 수 있음 | "영상 전체를 열기 전에 어떤 장면인지 작은 이미지로 먼저 보여줍니다." |
| 하이라이트 | 밤새 영상 중 볼 만한 핵심 장면 | 보호자 경험을 좋게 만드는 앱 기능 | "AI가 중요한 장면만 골라 보호자가 빠르게 확인하게 해줍니다." |
| Track A | 모든 움직임 클립을 빠르게 1차로 보는 분석 단계 | 싸고 빠르게 전체 영상을 훑는 역할 | "Track A는 밤새 찍힌 장면을 먼저 빠르게 훑는 1차 검수자입니다." |
| Track B | 애매하거나 중요한 장면을 더 자세히 보는 정밀 분석 단계 | 정확도를 높이고 놓친 행동을 회복하는 역할 | "Track B는 의심 가는 장면만 확대해서 다시 보는 정밀 분석자입니다." |
| Zero-shot VLM | 따로 도마뱀 전용 학습을 많이 시키지 않아도 영상을 이해하는 범용 AI | 초기 제품에서 빠르게 행동 분석을 시작할 수 있음 | "처음 보는 영상도 이해할 수 있는 범용 시각 AI를 1차 분석에 씁니다." |
| VLM | Vision-Language Model. 이미지나 영상을 보고 말로 설명할 수 있는 AI | 펫캠 영상을 이해하는 기본 두뇌 | "VLM은 영상을 보고 '무슨 일이 일어났는지' 말로 판단하는 AI입니다." |
| 멀티모달 VLM | 영상, 이미지, 글 같은 여러 종류의 정보를 함께 보는 AI | 영상만 보는 것보다 상황 이해가 좋아짐 | "영상과 주변 정보를 같이 읽는 AI라서 더 사람처럼 판단할 수 있습니다." |
| SegmentVLM | 긴 영상을 짧은 조각으로 나눠서 자세히 보는 분석 방식 | 미세 행동을 더 잘 잡아냄 | "SegmentVLM은 긴 영상을 통째로 보지 않고, 중요한 몇 초만 잘라서 자세히 보는 방식입니다." |
| event segment | 행동이 일어난 짧은 장면 조각 | 60초 전체보다 5~15초 핵심 장면이 판단에 유리함 | "AI가 전체 영상을 보는 대신 행동이 일어난 짧은 구간만 집중해서 봅니다." |
| 행동 라벨 | AI가 붙이는 행동 이름 | drinking, eating, moving처럼 결과가 구조화됨 | "AI가 장면마다 '물 마심', '먹이 반응', '이동' 같은 이름표를 붙입니다." |
| top-1 행동 라벨 | AI가 가장 가능성이 높다고 고른 1등 행동 | 앱에 표시할 기본 결과가 됨 | "AI가 여러 후보 중 가장 그럴듯한 행동 하나를 대표 결과로 저장합니다." |
| confidence | AI가 자기 판단을 얼마나 확신하는지 표시한 점수 | 낮으면 Track B나 사람 검수로 넘길 수 있음 | "AI가 자신 없는 장면은 스스로 표시하고 더 정밀한 분석으로 넘깁니다." |
| 판단 신뢰도 | confidence의 쉬운 표현 | 투자자에게는 이 표현이 더 이해하기 쉬움 | "AI가 답만 내는 게 아니라, 얼마나 확실한지도 함께 표시합니다." |
| ROI | Region of Interest. 관심 구역 | 물그릇, 먹이그릇, 은신처 같은 중요한 위치 | "AI가 화면 전체를 막연히 보는 게 아니라, 물그릇과 먹이그릇 같은 중요한 구역을 압니다." |
| motion metadata | 움직임에 대한 부가 정보 | 어디서, 언제, 얼마나 움직였는지 알려줌 | "단순히 영상만 보는 게 아니라 움직임의 위치와 패턴도 함께 씁니다." |
| metadata | 영상 주변의 설명 정보 | 시간, 위치, 개체, ROI 같은 판단 재료 | "AI에게 영상만 주지 않고, 판단에 필요한 주변 정보도 같이 줍니다." |
| Evidence Layer | AI 판단의 근거를 모아두는 층 | 왜 그렇게 판단했는지 설명 가능해짐 | "AI가 그냥 맞히는 게 아니라, 어떤 근거로 판단했는지 남깁니다." |
| structured context packet | AI에게 줄 판단 재료를 정리한 묶음 | 정확도를 올리는 다음 승부처 | "AI에게 '이 영상 봐봐'가 아니라, 시간, 위치, 움직임, 개체 정보를 묶어서 같이 줍니다." |
| key frame | 행동을 가장 잘 보여주는 대표 장면 | 보호자와 검수자가 빠르게 확인 가능 | "긴 영상 대신 결정적인 한 장면을 뽑아 판단 근거로 보여줍니다." |
| HITL | Human-in-the-loop. 사람이 AI 결과를 검수하는 방식 | 애매한 장면을 사람이 확인하고 데이터 품질을 높임 | "AI가 애매해하는 장면은 사람이 검수하고, 그 결과가 다시 AI를 똑똑하게 만듭니다." |
| 사람 검수 | HITL의 쉬운 표현 | 투자자에게 더 직관적인 표현 | "AI가 전부 혼자 끝내는 게 아니라, 중요한 장면은 사람이 확인해 품질을 올립니다." |
| 데이터 루프 | AI 결과와 사람 검수가 다시 학습 자산으로 쌓이는 순환 구조 | 시간이 갈수록 우리 데이터가 진입장벽이 됨 | "사용할수록 검수 데이터가 쌓이고, 그 데이터가 다시 AI 성능을 올리는 구조입니다." |
| 평가셋 | AI가 잘 맞히는지 시험해보는 정답 모음 | 정확도 주장의 근거 | "저희는 감으로 좋다고 말하는 게 아니라, 정답 데이터로 성능을 확인합니다." |
| 프롬프트 v4.0 | AI에게 어떤 기준으로 판단하라고 알려주는 자체 지침 | 같은 모델을 써도 결과 품질을 통제하는 핵심 | "프롬프트는 AI에게 행동 판단 기준을 알려주는 저희의 판독 매뉴얼입니다." |
| 과잉 해석 방지 | 애매한 장면을 먹었다, 마셨다 식으로 쉽게 단정하지 않는 것 | 오판을 줄이는 핵심 기준 | "그릇 근처에 있었다고 바로 먹었다고 하지 않고, 실제 혀 접촉 같은 증거가 있어야 판단합니다." |
| fine-tuned model | 우리 데이터로 추가 학습한 전용 모델 | 범용 AI보다 도마뱀 행동을 더 잘 볼 수 있음 | "남들이 쓰는 범용 AI 위에, 저희 데이터로 학습한 도마뱀 특화 모델을 얹습니다." |
| 도마뱀 행동 특화 모델 | fine-tuned model의 쉬운 표현 | 장기적으로 비용과 정확도를 모두 개선 | "우리 서비스에서 쌓인 라벨 데이터로 도마뱀 행동에 특화된 모델을 만듭니다." |
| rules | 행동 판단 규칙 | AI가 실수하기 쉬운 부분을 도메인 지식으로 보완 | "AI가 헷갈리는 부분은 사육 지식 기반 규칙으로 한 번 더 거릅니다." |
| ensemble | 여러 판단기의 결과를 합쳐 최종 판단하는 방식 | 한 모델에만 의존하지 않음 | "범용 AI, 특화 모델, 규칙을 합쳐서 최종 결정을 내립니다." |
| 단일 벤더 종속 | 특정 AI 회사 모델 하나에만 의존하는 상태 | 비용, 정책, 성능 변화에 취약함 | "저희 구조는 특정 외부 AI 하나에 묶이지 않도록 설계되어 있습니다." |
| 외부 AI 의존도 | 외부 모델을 얼마나 많이 쓰는지 | 줄일수록 비용 통제와 기술 자산화에 유리 | "초기에는 외부 AI를 활용하되, 데이터가 쌓이면 자체 모델 비중을 높여 비용을 낮춥니다." |
| 분석 단위 비용 | 영상 하나 또는 클립 하나를 분석하는 데 드는 비용 | 대규모 서비스에서 수익성과 직결됨 | "AI 분석 비용을 클립 단위로 통제해야 서비스가 커져도 마진이 남습니다." |
| Production Scale | 실제 고객이 많이 써도 돌아가는 운영 단계 | 실험실 기술에서 사업 기술로 넘어가는 단계 | "베타를 넘어 실제 고객 수가 늘어도 안정적으로 분석되는 구조입니다." |
| 해자 | 경쟁사가 쉽게 따라오기 어려운 장벽 | 투자자 관점에서 매우 중요한 개념 | "저희 해자는 카메라가 아니라, 시간이 갈수록 쌓이는 행동 데이터와 판단 규칙입니다." |

| 용어 | 쉽게 말하면 | 투자자에게 설명 |
|---|---|---|
| RBA 1.0 | 현재 베타 단계의 기본 행동 분석 엔진 | Track A와 Track B로 핵심 행동을 자동 분류하고 사람 검수로 품질을 올리는 단계 |
| RBA 1.5 | 확실한 건 빠르게 처리하고 애매한 것만 정밀 분석하는 단계 | 모든 장면에 비싼 AI를 쓰지 않고, 중요한 장면에 집중해 비용과 정확도를 같이 잡는 단계 |
| RBA 2.0 | 대규모 운영과 자체 특화 모델로 가는 단계 | 외부 AI 의존도를 낮추고, 우리 데이터와 모델을 자산화하는 단계 |

| 다종 확장 | 크레스티드 게코 외 다른 파충류·양서류까지 넓히는 계획 | 한 종에서 쌓은 분석 구조를 다른 종으로 확장해 시장을 넓히는 전략 |

## 투자자에게 특히 쉽게 풀어야 하는 말

### 1. "이게 그냥 알고리즘이 아니라 AI인가?"

쉽게 답하면:
"단순 알고리즘은 정해진 조건만 확인합니다. 예를 들어 '움직임이 있으면 알림' 정도입니다. RBA는 영상 속 장면을 보고 이것이 물 마심인지, 먹이 반응인지, 단순 이동인지 의미를 판단합니다. 그래서 단순 제어 알고리즘이 아니라 영상 이해 AI에 가깝습니다."

더 짧게:
"움직임 감지는 알고리즘이고, 움직임의 의미를 해석하는 게 RBA AI입니다."

### 2. "왜 그냥 CCTV 녹화랑 다른가?"

쉽게 답하면:
"CCTV는 영상을 쌓아두기만 합니다. RBA는 밤새 영상 중 중요한 장면을 골라서, 어떤 행동이 있었는지 시간표와 요약으로 바꿔줍니다."

더 짧게:
"녹화가 아니라 해석입니다."

### 3. "왜 큰 AI 회사가 바로 따라오지 못하나?"

쉽게 답하면:
"범용 AI는 영상을 볼 수 있지만, 도마뱀 사육 상황의 판단 기준과 실제 라벨 데이터는 없습니다. 저희는 물그릇, 먹이그릇, 은신처 위치, 행동 규칙, 사람 검수 데이터가 쌓이기 때문에 시간이 지날수록 따라 하기 어려워집니다."

더 짧게:
"AI 모델 자체보다, 도마뱀 행동 데이터와 판단 규칙이 해자입니다."

### 4. "정확도 89%면 충분한가?"

쉽게 답하면:
"1차 분류기로만 보면 한계가 있습니다. 그래서 RBA는 자신 없는 장면을 Track B로 넘겨 정밀 분석하고, 애매한 장면은 사람 검수로 다시 학습 데이터화합니다. 목표는 단순 정확도 숫자보다 실제 보호자가 믿고 쓸 수 있는 밤사이 행동 해석입니다."

더 짧게:
"한 번에 맞히는 AI가 아니라, 애매한 장면을 다시 보고 좋아지는 AI 구조입니다."
