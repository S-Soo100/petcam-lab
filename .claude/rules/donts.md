# Don'ts — petcam-lab 전역 금지 규칙

> 반복되는 실수를 방지하기 위한 하드 룰. 이 파일은 60줄 이내 유지, 기능별 세부 룰은 `donts/` 하위 파일로 분리.
> 출처: `tera-ai-product-master/.claude/rules/donts.md` 제너럴 섹션에서 이 레포 맥락(Python/FastAPI/영상)에 맞게 발췌.

## 🔁 새 룰 추가 기준 (Three-Strike Rule)

| 발생 횟수 | 조치 |
|---------|------|
| **1회** | `~/.claude/projects/-Users-baek-petcam-lab/memory/feedback_*.md` 메모리에만 저장 |
| **2회째** | `.claude/donts-audit.md`에 **"승격 후보"** 플래그 + 맥락 메모 |
| **3회째** | 이 파일 또는 `donts/{기능}.md`에 정식 룰로 추가 + 메모리에 "donts 승격됨" 표시 |

**재발 판정:** 동일 feedback 파일 재트리거뿐 아니라 **비슷한 맥락**(다른 파일·다른 기능이어도 원인·패턴이 같음)도 포함.
**예외 — 1회에도 즉시 추가:** 사용자가 "다시는 하지 마" 명시 / 되돌리기 비용이 매우 큰 사고(DB 삭제, 영상 소실 등).

## 🚫 제너럴 (모든 작업)

1. **기억으로 단정 금지** — 파일·설정·라이브러리 API를 언급하기 전에 반드시 도구로 확인. 특히 OpenCV/FastAPI 시그니처는 `Read`/공식 문서로 검증한 뒤 말한다.
2. **최소 변경 원칙** — 요청 범위 밖의 리팩토링·스타일·과잉 수정 금지. "1~3줄로 끝낼 수 있나?"부터 점검.
3. **진단 없는 수정 금지** — 버그 보고 받으면 "로그 + 코드 추적 + `git diff`" 3단 진단 → 그다음에 수정. 추측으로 코드부터 고치지 않는다.
4. **자체 검수 전 "테스트해보세요" 금지** — 수정 후 흐름 추적 + 엣지케이스 점검을 직접 한 뒤에만 사용자에게 넘긴다.
5. **대규모 변경 통째로 읽지 않기** — 변경 파일 10개+는 `git diff --stat` → 그룹핑 → 순차 처리. 디렉토리 통째 읽기 금지.
6. **같은 모델 자기 리뷰 금지** — 검수는 다른 모델 교차 리뷰(Gemini ↔ Claude ↔ Codex). 자기 동의 편향 때문.
7. **기존 구조 보존 기본값** — 폴더 구조·네이밍·인터페이스는 "바꿔달라" 명시 없는 한 유지. 전면 재작성은 합의 후에만.
8. **AI 생성 설계 맹신 금지** — 외부 AI가 만든 스펙/코드는 초안. 비판적 검수 후 선택 반영.
9. **파괴적 git 작업 금지** — `reset --hard`, `push --force`, `branch -D`, `clean -f`는 사용자 명시 승인 없이 금지. 먼저 원인 조사.
10. **병렬 에이전트 경계 준수** — 동시 투입 시 각자 "새 파일만" 쓰게 하고, 같은 파일 수정 금지. 끝나면 자체 검수.
11. **비밀값 커밋 금지** — RTSP URL의 비밀번호·Supabase 키·Tapo 계정은 `.env`에만. `.env.example`은 더미 값으로만.
